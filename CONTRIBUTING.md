# Contributing

Contributions are welcome! Anyone can help improve this project.

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b my-feature`)
3. Commit your changes
4. Push to the branch (`git push origin my-feature`)
5. Open a Pull Request

## What You Can Contribute

- Performance optimizations and benchmarks on DGX Spark
- Bug fixes in launch scripts or Dockerfiles
- Documentation improvements
- New kernel implementations or quantization strategies
- Support for other Qwen models or hardware configurations

## Guidelines

- Keep PRs focused — one feature or fix per PR
- Include benchmark results if your change affects performance
- Test your changes on DGX Spark if possible
- Follow existing code style
